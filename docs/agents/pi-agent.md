# Pi Agent

Pi is the leanest of the three agents in this repository and has strong customization potential.

## Why Pi Is Useful

- simple model/provider configuration
- straightforward launcher patterns
- good backup coding workflow when you want less ceremony

## Recommended Commands

- `pidev`
- `psafe`
- `pqcoder`
- `pplan`
- `pheavy`
- `ptito`
- `pmodels`
- `pmodel <model> [path]`

## Best Use Cases

- lightweight coding sessions
- secondary coding workflow
- testing custom provider mappings to local Ollama models

## Continue.dev Pairing

Pi works well as part of a broader local stack where Continue.dev handles editor-native chat and autocomplete.

Practical split:

- Pi -> explicit agent workflow and task execution
- Continue.dev -> editor-integrated chat and autocomplete
- CodeGemma -> autocomplete-oriented local model

## Current Model Role Mapping

- Jan-code -> daily coding default
- Aikid Qwen3-coder -> fast code and chat with thinking
- Qwen3.5 Opus 4B -> complex thinking code and heavier work
- Orlex and CodeTito -> backup models
- CodeGemma -> autocomplete-oriented helper, not the main Pi chat model
